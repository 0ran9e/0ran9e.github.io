K-means Clustering

我们可以先从一个2D的数据集开始，它可以帮助理解K-mean算法的工作原理。之后我们再使用K-means算法进行图像压缩，方法是将在图像中出现的颜色的数量减少到仅在图像中最常见的颜色数量，即只显示出现出现频率最高的几个颜色。

Implementing K-means

K-means算法是一种将相似数据示例自动聚类在一起的方法。具体而言，给我们一个数据集 ${x^{(1)},…,x^{(m)}}$ ，并且希望将数据分组为几个”聚类”。K-means背后的含义是一个迭代的过程，它通过猜测初始质心开始，然后根据赋值重新计算质心来重新估计此猜测。

K-means算法如下：

算法的内部循环重复执行两个步骤：(i) 将每个训练样例分配给其最接近的质心。 (ii) 通过分配的点重新计算每个质心的平均值。K-means算法将总是收敛到一些最终形式的质心的集合，需要注意的是，收敛的方法可能并不总是理想的，它取决于质心的初始设置。因此在实践中，K-means算法通常会在不同的随机初始化下运行几次，并选择成本函数值（实真）最小的方案。

接下来我们将分别实现K-means算法的两个阶段。

Finding closest centroids

在K-means算法的聚类分配阶段，给定当前质心的位置，该算法将每个训练实例 $x^{(i)}$ 分配到最接近的质心，具体来说，对于每个实例 i ：
$$
c^{(i)} := j \quad that \ minimizes \quad ||x^{(i)}-µ{j}||^2
$$
其中 $c^{(i)}$ 是最靠近 $x^{(i)}$ 质心的索引，$µ{j}$ 是第 j 个质心的位置（值）。注意 $c^{(i)}$ 对应于起始代码中的 idx(i) 。

我们的任务是完成 findClosestCentroids.m 中的代码，该函数用数据矩阵 X 和所有质心的位置，返回一组保存着最接近质心索引的一维数组。

Computing centroid means

算法的第二阶段会为每个质心重新计算分配给它的点的平均值，具体而言，对于每个质点，我们设置：
$$
μ_k := \frac{1}{|C_k|}\sum_{i∈C_k}x^{(i)}
$$
其中 $C_k$ 是分配给质心k的一组实例，举例来说，如果两个实例 $x^{(3)}$ 和 $x^{(5)}$ 被分配给质心 k=2，那么就应该更新$μ_2 = \frac{1}{2}(x^{(3)}+x^{(5)})$ 。

在 computeCentroids.m 中完成代码，我们既可以在质心上循环或示例中循环来实现此功能，但如果我们可以用矢量化实现的话，那么我们的代码可以运行的更快。

Image compression with K-means

我们将要用K-means算法实现图片压缩。在图像的24位色彩表示中，每个像素都表示位三个8位无符号整数（范围从0到255），用于指定红色，绿色和蓝色强度值，这种编码通常被称为RGB编码。我们的图像包含数千种颜色，在这此练习中，我们将减少到16种颜色。

通过这种方法，我们可以有效地压缩图片。具体来说，我们只需要存储16种选定颜色的RGB值，并且对于图像中的每个像素，我们只需要存储该位置的颜色索引（只需要4位即可表示16种可能）。

在本练习中，我们将使用K-means算法来选择将用于表示压缩图像的16种颜色。我们可以把原始图像中的每个像素作为一个数据示例，并使用K-means算法来查找16种颜色，以最佳的方式对三维RGB空间中的像素进行分组。一旦在图像上计算出了集群的质心，我们将使用这16种颜色来替换原始图像种的像素。

Principal Component Analysis

我们将使用主成分分析（PCA）来执行降维。我们将首先尝试使用2D数据集来直观了解PCA的工作原理，然后将其用于5000张面部图像数据集的更大数据集。

Example Dataset

文档为了邦族我们理解PCA的工作原理，首先给了我们一个2D数据集，该数据集的一个方向变化较大，另一变化较小，脚本 ex7_pca.m 将绘制训练数据。在本练习的这一部分中，我们将看到数据从2D减小到1D时发生的情况。实际上，我们可能想要将数据从256个减少到50个，但在这个例子中使用较低维度的数据可以让我们更好的可视化算法。

Implementing PCA

PCA由两个计算步骤组成：首先，计算数据的协方差矩阵。然后使用MATLAB的 SVD 函数来计算特征向量 $U_1, U_2, …, U_n$ 。这些将对应于数据变化的主要组成部分。

在使用PCA之前，先要通过从数据集中减去每个特征的平均值来对数据进行归一化，并对每个维度进行缩放以使它们在相同的范围内。在提供的脚本 ex7_pca.m 中，使用 featureNormalize 函数为我们进行了规范化。

标准化数据后，我们可以运行PCA来计算主要成分。我们的任务时完成 pca.m 中的代码来计算数据集的主要组成部分。首先我们应该计算数据的协方差矩阵，它由下式给出：
$$
Σ = \frac{1}{m}X^TX
$$
其中 X 是行为示例的数据矩阵，m 是示例的数量。请注意 Σ是一个n×n的矩阵，而不是求和运算符 。

在计算协方差矩阵后，我们可以对其运行 SVD 来计算主成分。在MATLAB中，可以使用这条命令来运行 SVD ：[U, S, V] = svd(Sigma)，其中 U 包含主要成分，S 包含对角矩阵。

完成 pca.m 后，ex7_pca.m 脚本将在示例数据集上运行PCA，并绘制找到的相应的主成分（特征向量），由于 $U_1$ 和 $U_2$ 对于第一个主成分是同样有效的选择，因此MATLAB可能会输出负值。

Dimensionality Reduction with PCA

计算主成分后，可以用它们将每个示例的特征向量投影到低维空间，$x^{(i)}→z^{(i)}$ 。在这部分练习中，我们将使用PCA返回的特征向量并将示例的数据集投影到一维空间。

在实践中，如果我们使用的是线性回归或神经网络等学习算法，则可以用投影后的数据而不是原始数据，通过使用投影数据，可以更快的训练模型。因为输入的维度更少。

Face Image Dataset

在这部分练习中，我们将在面部图像上运用PCA以了解如何在实践中使用它来减少尺寸。数据集 ex7faces.mat 包含了灰度级为 32×32 面部图像的数据集。 X 的每一行对应着一个面部图像（一行的向量长度为1024）。ex7_pca.m 将会可视化前100张面部图像并显示出来。

PCA on Faces

为了在面部数据集上运行PCA，我们首先通过从数据矩阵X中减去每个特征的平均值来标准化数据集，ex7_pca.m 已经帮我们实现好了，然后就运行我们的PCA代码。运行PCA后，我们将得到数据集的主要组件。请注意U（每一行 ）中的每个主成分都是一个长度为n的向量（其中对于人脸数据 n=1024 ）。事实证明，我们可以通过将每一个重塑成与原始数据集中的像素相应的 32×32 矩阵来将这些主要组件可视化。ex7_pca.m 显示描述最大变化的前36个主要组件，如果有需要还可以更改代码以显示更多主要组件，以查看它们如何捕获越来越多的细节。

Dimensionality Reduction

现在我们已经计算了面部数据集的主要组件，我们可以使用它来减少面部数据集的维度。这样我们就可以用较小的输入尺寸而不是原始的1024尺寸的学习算法，这样可以帮助加快我们的学习算法。

ex7_pca.m 的下一部分只将面部数据集投影到前100个主要组件上。也就是说，现在通过矢量 $z^{(i)}∈\mathbb{R}^{100}$ 来描述每个人脸图像。

要了解降维中丢失了什么，可以仅使用投影数据集恢复数据。在 ex7_pca.m 中，执行数据的近似恢复，并将原始和投影的脸部图像并排显示。从重建中，我们可以观察到在细节丢失的情况下，脸部大体结构和外观都保留下来了。数据集大小的显著减小（超过10倍）可以帮助加快我们学习算法的速度。例如，我们正在训练一个神经网络来执行人员识别（获取人脸图像，预测该人员身份），则可以使用缩小尺寸到仅为100个维度而不是原始像素。
